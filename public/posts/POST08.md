O que são Heurísticas?
São atalhos mentais ou regras práticas de bom senso que ajudam a resolver problemas de usabilidade sem precisar "reinventar a roda". Aplicá-las pode eliminar muitos dos erros de usabilidade em produtos digitais.

1. **Visibilidade do status do sistema:** O sistema deve sempre manter o usuário informado sobre o que está acontecendo (ex: barras de carregamento, spinners);
2. **Correspondência entre o sistema e o mundo real:** Use uma linguagem e conceitos familiares ao usuário, em vez de termos técnicos ou códigos de erro (ex: dizer "página não encontrada" em vez de apenas "Erro 404");
3. **Controle e liberdade do usuário:** Ofereça saídas claras, como botões de "desfazer", "cancelar" ou "voltar atrás", para que o usuário não se sinta preso;
4. **Consistência e padrões:** Não tente ser excessivamente criativo com padrões estabelecidos. Se um botão de salvar é azul em uma tela, ele deve ser azul em todo o sistema;
5. **Prevenção de erros:** É melhor evitar que o erro ocorra do que exibir uma mensagem depois. Use confirmações antes de ações destrutivas, como deletar um arquivo;
6. **Reconhecimento em vez de memorização:** Minimize a carga de memória do usuário tornando objetos, ações e opções visíveis (ex: usar ícones de lixeira para excluir, sugestões de busca automática);
7. **Flexibilidade e eficiência de uso:** O sistema deve ser simples para iniciantes, mas oferecer atalhos e aceleradores para usuários experientes (experts);
8. **Design estético e minimalista:** Diálogos e telas não devem conter informações irrelevantes. "Menos é mais" para evitar confusão visual;
9. **Ajude os usuários a reconhecer, diagnosticar e recuperar-se de erros:** As mensagens de erro devem ser claras, indicar o problema e sugerir uma solução construtiva;
10. **Ajuda e documentação:** Embora o ideal seja um sistema fácil de usar sem manuais, é necessário fornecer uma seção de ajuda ou FAQ para quando o usuário precisar de suporte.
